\end{DoxyEnumerate}$<$/details$>$\hypertarget{md_README_autotoc_md0}{}\doxysubsection{About The Project}\label{md_README_autotoc_md0}
In the course of this Lab assignment, students are going to implement a minimal BDD package in C++. This package implements the fundamental manipulation methods for ROBDDs as they were introduced in the lecture {\itshape Verification of Digital Systems} by Prof. Kunz. The package will be implemented using the Test Driven Development (TDD) paradigm presented by Dr. Wedler.

The project is split into three parts. 

(\href{\#top}{\texttt{ back to top}})\hypertarget{md_README_autotoc_md1}{}\doxyparagraph{Part1\+:}\label{md_README_autotoc_md1}
Implementation of the basic functionality of the BDD package using the TDD methodology. This is the biggest part of the project.

Main tasks in this part\+:
\begin{DoxyItemize}
\item Set up and maintain a Git repository
\item Use CMake as a build system of the project
\item Verify the code using GTest
\item Set up a Continuous Integration (CI) pipeline with Git\+Hub Actions $\ast$$\ast$(Mandatory)$\ast$$\ast$
\item Use Doxygen (or a similar tool) to generate documentation for the API. \+\_\+(\+Optional)\+\_\+ 
\end{DoxyItemize}

(\href{\#top}{\texttt{ back to top}})\hypertarget{md_README_autotoc_md2}{}\doxyparagraph{Part2\+:}\label{md_README_autotoc_md2}
Improvement of the performance of the implementation via provided benchmarks.

Main tasks in this part\+:
\begin{DoxyItemize}
\item Learning how to identify performance bottlenecks within the code.
\item Analyzing benchmark results.
\item How to overcome bottlenecks. 
\end{DoxyItemize}

(\href{\#top}{\texttt{ back to top}})\hypertarget{md_README_autotoc_md3}{}\doxyparagraph{Part3\+:}\label{md_README_autotoc_md3}
Extending the existing implementation by a practical application of BDD. Using BDDs, it is possible to symbolically represent a state-\/space. This representation allows to check quickly, whether a specific state is within the reachable state space or not.
